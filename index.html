GAME FLOW
From login to payout
1
REGISTER / LOGIN
User creates account with username, email, Ghana phone, and password. JWT issued on success. Token stored in localStorage, socket connects automatically.
POST /api/auth/login → { token, user }
2
DEPOSIT FUNDS
Player deposits via MTN MoMo, Vodafone Cash, AirtelTigo, or Stripe card. Balance updated immediately on webhook confirmation. Push notification sent.
POST /api/payments/deposit → MoMo prompt
3
PLACE A STAKE
Player enters stake amount (GH₵1 min). Stake deducted from balance optimistically. Player enters matchmaking queue for their stake tier.
socket.emit('matchmaking:join', { stake: 50 })
4
OPPONENT MATCHED
Server finds another player in the same stake range within 30s. Both players receive game:start with opponent name and game ID. Push notification fires if app is in background.
socket.on('game:start', { gameId, opponentName, stake })
5
TAP THE BUZZER
60 seconds of frantic tapping. Every tap emits to the server which validates and increments score. Opponent's live score shown. Premium players activate multipliers.
socket.emit('game:tap') → game:score_update
6
RESULT + PAYOUT
Server calculates winner server-side. Winner receives 2× their stake. Tie returns both stakes. Result modal shows scores. Confetti on win. Share button triggers native share sheet.
socket.on('game:over', { won, yourScore, newBalance })
7
WITHDRAW WINNINGS
Player withdraws to their Mobile Money account. Admin can monitor all transactions in the dashboard. Balance visible in top bar at all times.
POST /api/payments/withdraw → MoMo transfer
DEPLOYMENT
Ship in 3 commands
Runs on any Node.js host — Render, Railway, DigitalOcean, VPS. Needs a MySQL database and your .env credentials.
# 1. Install dependencies
$ npm install
 
# 2. Set up database tables
$ npm run migrate && npm run migrate:admin
 
# 3. Start the server
$ npm start
Requires: Node ≥18 · MySQL 8+ · .env with DB credentials + VAPID keys + payment API keys
BUZZERBET
FULL-STACK PWA · GHANA · BUILT ENTIRELY WITH CLAUDE
AUTHENTICATION
JWT-based auth with bcrypt password hashing. On login, a signed token is stored in localStorage and the Socket.io client connects immediately using it as a handshake credential.
Login & Registration — Username, email, Ghana phone number, password. Phone validated against GH format regex.
Auto-reconnect — On page load, existing token is verified via /api/auth/profile. User skips auth screen if session is valid.
JWT expiry handling — 401 responses clear the token, disconnect the socket, and return the user to the auth screen automatically.
Joi validation — All inputs validated server-side with detailed error messages returned to the UI.
LIVE GAME ARENA
The core game loop. Two players tap their golden buzzer as fast as possible for 60 seconds. The server is authoritative — all scoring happens server-side to prevent cheating.
Server-authoritative scoring — Every tap event is processed server-side. Score updates are sent back via game:score_update. Client shows optimistic increments for responsiveness.
Live opponent score — Opponent's tap count streamed via game:opp_score in real time. Shown in the top area of the arena.
Multipliers — Premium plans unlock ×2, ×3, ×4 multiplier buttons. Each usable once per game for 30 seconds. Server validates plan eligibility.
Disconnect handling — If a player disconnects mid-game, a 10s grace period is given for reconnect. If they don't return, the remaining player wins and receives the stake.
Offline tap buffering — Taps during a brief disconnect are stored in IndexedDB and sent as a batch when the socket reconnects.
ADMIN DASHBOARD
A full 7-page admin console for monitoring and managing the platform. Protected by admin role in JWT. Canvas-based revenue & activity charts with 7D/30D/90D toggles.
Dashboard — Revenue, active users, games today, premium subs. Revenue area chart + payment split donut. Live event feed updating every 3s.
Live Games — Active match table with real-time score counters and countdown timers. Admin can force-end any game.
Users — Searchable table. User detail modal with balance adjustment, messaging, and ban controls.
Transactions — Filterable by type (deposits, withdrawals, stakes, winnings, premium). Color-coded status badges.
Settings — Game config (duration, stake limits), premium pricing, maintenance mode toggles, platform health stats.
PROGRESSIVE WEB APP
Full PWA stack. Installable on Android and iOS home screens. Works offline with a cached app shell. Service worker handles caching, push events, background sync, and periodic refresh.
Install prompt — Custom "Add to Home Screen" banner appears automatically after 10s on eligible browsers. Dismissed state persists for 7 days.
Offline caching — App shell pre-cached on install. Stale-while-revalidate for static assets. Network-first for API calls with offline fallback.
Screen Wake Lock — Prevents screen from sleeping during an active game. Automatically released on game over or tab hide.
Push Notifications — VAPID web push for game matched, win/loss, deposit/withdrawal confirmation, premium expiry alerts.
Haptic Feedback — Vibration API called on every tap (8ms) and victory pattern on win. Opt-in based on device support.
PAYMENTS + PREMIUM
Ghana-native payment integrations for deposits and withdrawals, plus Stripe for international card payments. Premium subscription tiers gate multiplier features.
MTN Mobile Money — Prompt-to-pay via MTN MoMo API. Webhook confirms payment before balance credit.
Vodafone Cash + AirtelTigo — Same prompt-to-pay flow for Voda and Airtel providers. All three support deposits and withdrawals.
Stripe — Card payments via Stripe PaymentIntents. Webhooks confirm charges before balance is updated.
Premium tiers — Silver (GH₵29), Bronze (GH₵49), Gold (GH₵79). Monthly. Unlock ×2, ×3, ×4 in-game multipliers respectively.
Transaction history — All deposits, withdrawals, stakes, winnings, and premium purchases logged in DB and visible in-app and in admin.
GAME RESULT
Post-game modal showing the outcome with full score breakdown, payout amount, and social sharing. Three distinct states: win, loss, and draw.
Win state — Green header, confetti particle burst, vibration pattern, "+GH₵X" payout display. Balance updated server-side and reflected immediately.
Loss state — Red header, "-GH₵X" stake shown. Encourages retry. No balance change (stake already deducted at game start).
Draw state — Gold header, "±0" shown, both stakes returned server-side to balance.
Share button — Triggers the Web Share API for native OS share sheet. Falls back to clipboard copy on unsupported browsers.
LEADERBOARD & HISTORY
Global leaderboard ranking players by total wins. Current user is highlighted in green. Game history shows your last 20 matches with opponent name, scores, stake, and payout.
Leaderboard — Top players ranked by total wins. 🥇🥈🥉 icons for top 3. Your row highlighted. Loaded from GET /api/games/leaderboard.
Game History — Last 20 matches for the logged-in user. Shows WIN/LOSS/DRAW badge, opponent name, tap scores, stake amount, and net payout.
Accessible from menu — Both screens are overlays accessible from the hamburger menu. Loaded lazily on open.
